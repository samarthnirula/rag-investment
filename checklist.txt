SECTION 2 — KNOWN GAPS (be honest about these in the demo)
These cannot be fixed tonight. Prepare verbal answers explaining the gap and
what you would build to close it.
--------------------------------------------------------------------------------

[ ] GAP 1: Vision/OCR for charts, bar graphs, logos, and map graphics
    Affects: Q15, Q16, Q18
    What to say: "Text extraction cannot read bar charts, logo images, or
                  spatial map data. The fix is to pass each page as an image
                  to a vision model (Claude claude-sonnet-4-6 vision) during ingestion
                  and store the vision-extracted text alongside the PDF text.
                  This is the most impactful single upgrade for this document type."

[ ] GAP 2: Footnote-as-truth-source (when footnote contradicts chart body)
    Affects: Q7, Q8
    What to say: "The chunker preserves footnote text in the chunk, so the
                  information is there. But the system has no way to know that
                  a footnote overrides the chart headline. The fix is to tag
                  footnote lines during parsing and give them elevated weight
                  in the prompt context."

[ ] GAP 3: Intra-document inconsistency detection
    Affects: Q5 (5,500 vs 5,000 on different pages of same document)
    What to say: "The prompt instructs Claude to show conflicting numbers
                  separately, but retrieval only returns top-K chunks so it
                  may not surface both pages. The fix is to run a secondary
                  retrieval pass specifically looking for the same entity on
                  different pages of the same document."

[ ] GAP 4: Qualifier-aware chunking
    Affects: Q10 (65M SF including development vs 65M SF under ownership)
    What to say: "The chunker keeps each slide intact so the qualifier text
                  is in the chunk. But embedding compression means the vector
                  for both chunks looks similar. The fix is explicit scope
                  tagging at ingestion time."

[ ] GAP 5: Methodology-change detection in footnotes
    Affects: Q8 (DLR credit quality — top-10 vs top-100 customers)
    What to say: "This is the hardest class of RAG failure. The number looks
                  the same (50-51%) but the measurement changed. No text-based
                  system catches this without a structured fact-extraction layer
                  that parses metric definitions, not just metric values."
